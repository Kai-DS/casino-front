## 1. コンセプト

NEON JACKは、ジャグラー系スロットの「光ればボーナス」という分かりやすさをベースに、BIG後の **NEON COUNTDOWN** と、ボーナスループ型の **MIDNIGHT RUSH** を追加したネオンカジノ風Webスロットである。

通常時はシンプルに、JACK LAMPが光ればボーナス確定。BIG後はNEON COUNTDOWNでRUSH突入を抽選し、RUSH中はボーナス消化と判定をループする。

本機は実機の完全再現ではなく、ジャグラー風の告知ゲーム性をベースに、Webゲームとして遊びやすく仕上げたオリジナルスペックである。

## 2. 基本ゲーム性

- 通常時は3BET固定
- レバーONで内部抽選
- JACK LAMP点灯でボーナス確定(BIG/REGの種別は次ゲームで判明)
- ボーナスはBIGとREGの2種類
- 通常時BIG後はNEON COUNTDOWNへ移行
- 通常時REG後は通常時へ戻る
- RUSH中REG後は通常時へ戻らず、1G連判定またはRUSH継続抽選フェーズへ移行
- NEON COUNTDOWN成功でMIDNIGHT RUSHへ突入
- RUSH中はボーナス消化と継続判定をループ
- 通常時500G到達でBIG確定
- BIG・REGどちらでも天井カウンタはリセット
- RUSH終了後は通常時0Gから再開する

### 5-1. リール制御方針

- Webゲームとして、目押し不要の自動制御を採用
- 通常時は内部抽選に応じて小役を自動停止
- ボーナス入賞ゲームでは、適当押しでもBIGは7・7・7、REGは7・7・BARが揃う
- 演出として7やBARを狙う見せ場は残す
- 目押しができないプレイヤーでも損をしない仕様とする

### 5-2. リール配列に関する補足

本機はWebゲームとして、内部抽選結果に応じた自動停止制御を採用する。

そのため、リール配列は主に見た目・演出・図柄配置の雰囲気を決めるものであり、プレイヤーの目押し精度によって払い出しやボーナス入賞が失敗する仕様にはしない。

ボーナス入賞ゲームでは、BIGなら7・7・7、REGなら7・7・BARを自動的に表示する。

## 6. BET仕様

| 状態 | BET枚数 | 所持コイン減算 |
|---|---:|---|
| 通常時 | 3BET固定 | あり |
| ボーナス入賞ゲーム | 0BET | なし(演出扱い) |
| BIG中 | 3BET固定 | あり |
| REG中 | 3BET固定 | あり |
| REPLAY成立後の次ゲーム | 3BET自動 | なし(再遊技) |
| NEON COUNTDOWN中 | BET不要 | なし |
| RUSH継続抽選フェーズ | BET不要 | なし |

### 6-1. ボーナス入賞ゲームのBET扱い

ボーナス当選後のボーナス入賞ゲームは、Webゲームとしての遊びやすさを優先し、**演出上の入賞表示**として扱う。

- 追加BETは消費しない
- 所持コインは減算しない
- リール回転と図柄揃いの演出のみ行う

これは、実機のように毎ゲーム3BETを消費させる作りにすると、入賞表示のためだけにコインを使うことになり、Webゲームとして不自然になるため。

### 6-2. ボーナス中のBET処理

BIG中・REG中は、各ゲーム開始時に3BETを通常通り消費する。

| ボーナス | 払い出し総枚数 | 消費BET総枚数 | 純増 |
|---|---:|---:|---:|
| BIG | 168枚(14枚 × 12G) | 36枚(3枚 × 12G) | 132枚 |
| REG | 56枚(14枚 × 4G) | 12枚(3枚 × 4G) | 44枚 |

ボーナス中の純増は1Gあたり+11枚として扱う。

実装時、ボーナス中のBET減算を忘れると、BIG純増168枚、REG純増56枚と誤計算され、機械割が大きく上がるため注意。

## 10. ボーナス確率の定義

設定別のBIG確率・REG確率は、単独当選と同時当選をすべて含めた最終的な実質確率とする。

### 10-1. BIG確率に含めるもの

- 単独BIG
- CHERRY + BIG
- 中段CHERRY + BIG

天井BIGは通常抽選とは別扱い。

### 10-2. REG確率に含めるもの

- 単独REG
- CHERRY + REG

### 10-3. 注意点

実装時に、設定別BIG確率とは別にCHERRY + BIGなどを追加すると、想定よりボーナス確率が軽くなり、機械割が上がりすぎる。

各フラグの個別確率を設定する際は、最終的なBIG確率・REG確率が設定表の数値になるように調整する。

### 10-4. 分配の考え方(修正版)

| フラグ | 割合 |
|---|---:|
| 単独BIG | BIG全体の約75% |
| CHERRY + BIG | BIG全体の約22% |
| 中段CHERRY + BIG | BIG全体の約3% |
| 単独REG | REG全体の約75% |
| CHERRY + REG | REG全体の約25% |

#### 10-4-1. 中段CHERRY + BIGの整合性

旧版で「BIG全体の約5%」としていた配分を、中段CHERRY出現率目安(設定1で約1/8192)との整合性のため約3%に修正した。

設定1での検算:
```
1/218 × 3% = 約1/7267 (1/8192に近い)
旧 1/218 × 5% = 約1/4360 (1/8192より大幅に軽い)
```

中段CHERRY出現率を厳密に各設定の目安値に合わせる場合は、設定ごとに中段CHERRY確率を個別固定し、残りを単独BIGとCHERRY + BIGで按分する実装方針を取る。

### 10-5. BIGフラグの内部管理

実装上、BIGフラグは以下の3種類に分けて管理する。BIG終了後の遷移先が異なるため。

| フラグ名 | 該当 | BIG終了後の遷移 |
|---|---|---|
| NORMAL_BIG | 単独BIG、CHERRY + BIG、天井BIG | NEON COUNTDOWNへ |
| PREMIUM_BIG | 中段CHERRY + BIG | MIDNIGHT RUSH直行 |

REG側はすべて同じ扱い(通常時REG/RUSH中REGはbonusContextで区別、25章参照)。

## 11. 中段CHERRYの統一仕様

中段CHERRYは本機の最強プレミア役として、**発生する全ての場面で「その場面における最強の恩恵」に対応する**という統一ルールを採用する。

これにより、プレイヤーは「中段CHERRYが見えたら最強」という直感的な理解を全シーンで共有できる。

### 11-1. 場面別の中段CHERRYの意味

| 発生場面 | 中段CHERRYの意味 | 補足 |
|---|---|---|
| 通常時 | BIG + MIDNIGHT RUSH直行確定 | NEON COUNTDOWNをスキップ |
| ボーナス消化中 | 1G連確定 | 継続抽選フェーズをスキップ |
| RUSH継続抽選フェーズ | 継続確定 | 演出表示のみ、払い出しなし |
| NEON COUNTDOWN中 | 出現させない | COUNTDOWN自体が告知演出のため |

### 11-2. 通常時の中段CHERRY(プレミア役)

#### 11-2-1. 出現率目安

| 設定 | 出現率目安 |
|---:|---:|
| 1 | 1/8192 |
| 4 | 1/7281 |
| 5 | 1/6553 |
| 6 | 1/5957 |

#### 11-2-2. 恩恵

- BIG確定(PREMIUM_BIGフラグ)
- MIDNIGHT RUSH直行
- NEON COUNTDOWNはスキップ

#### 11-2-3. 内部処理

中段CHERRYは演出表示とする。内部的にはPREMIUM_BIGフラグ(CHERRY + BIGの特殊版)として扱う。

流れは以下の通り。

1. 中段CHERRY停止
2. 2枚払い出し
3. ボーナス入賞ゲームでBIG絵柄 7・7・7 を自動表示
4. BIG消化
5. 天井カウンタリセット
6. NEON COUNTDOWNをスキップしてMIDNIGHT RUSHへ直行

### 11-3. ボーナス消化中の中段CHERRY(1G連確定演出)

ボーナス消化中の中段CHERRY表示は、1G連抽選の特別演出パターンとして扱う。

#### 11-3-1. 出現条件

1G連抽選(全設定共通1/110)に当選したゲームのうち、さらに約1/30の確率で中段CHERRY演出として表示する。

これにより、ボーナス1Gあたりの中段CHERRY表示確率は約1/3300となる。

#### 11-3-2. 重要な設計原則

中段CHERRY演出は**1G連抽選の結果を見せる演出パターンの一つ**であり、1G連の発生率そのものを変えるものではない。

| 1G連抽選結果 | 表示パターン |
|---|---|
| 当選(1/110) | 通常の1G連演出 約96.7% / 中段CHERRY演出 約3.3% |
| 非当選 | 通常進行 |

これにより、17-2の平均1G連発生率約7.0%は維持される。

#### 11-3-3. 中段CHERRY演出時の払い出し

- ボーナス消化中の中段CHERRYは演出表示として扱い、専用の払い出しは行わない
- ボーナス図柄揃いによる14枚払い出しは通常通り実施
- 1G連確定として、ボーナス消化完了後に継続抽選フェーズをスキップ

### 11-4. RUSH継続抽選フェーズの中段CHERRY(継続確定演出)

継続抽選フェーズで表示される中段CHERRYは、内部継続フラグONの場合のみ出現する継続確定演出である。

詳細は19章を参照。

### 11-5. 中段CHERRY出現マトリクスまとめ

| 場面 | 出現可否 | 確率 | 恩恵 |
|---|---|---|---|
| 通常時 | あり | 11-2-1参照 | BIG + RUSH直行 |
| ボーナス中 | あり | 1G連経由で約1/3300 | 1G連確定 |
| 継続抽選フェーズ | あり(演出のみ) | 内部フラグONの一部 | 継続確定 |
| NEON COUNTDOWN中 | なし | - | - |

## 13. 通常時ゲームフロー

```text
通常時
 ↓
レバーONで内部抽選
 ↓
小役 or ハズレ → 通常継続
 ↓
ボーナス当選 → 第三リール停止後にJACK LAMP点灯
 ↓
液晶に「BONUS確定」を表示(種別は伏せる)
 ↓
次ゲーム = ボーナス入賞ゲーム(BET消費なし)
 ↓
リール回転 → 自動停止 → 第三リール停止時にBIG/REGが判明
 ├─ 7・7・7揃い → BIG確定表示
 └─ 7・7・BAR揃い → REG確定表示
 ↓
ボーナス消化
 ↓
NORMAL_BIG → NEON COUNTDOWNへ
PREMIUM_BIG → MIDNIGHT RUSH直行
NORMAL_REG → 通常時へ
```

### 13-1. 通常時の特徴

- 完全告知タイプ
- JACK LAMP点灯でボーナス確定
- BIG/REGの種別は次ゲームの図柄揃いで判明
- 目押し不要
- 通常時500G天井あり
- BIG・REGどちらでも天井カウンタをリセット

### 13-2. ボーナス告知からボーナス入賞までの流れ

通常時のボーナス当選ゲームでは、第三リール停止後にJACK LAMPを点灯させ、液晶に「BONUS確定」を表示する。**この時点では、BIGかREGかを画面上では明示しない。**

次ゲームをボーナス入賞ゲームとして扱い、リール停止演出によってボーナス図柄を自動停止させる。

第三リール停止時に、以下のいずれかの図柄組み合わせが揃うことで、プレイヤーにボーナス種別を告知する。

| 図柄組み合わせ | ボーナス種別 |
|---|---|
| 7・7・7 | BIG BONUS |
| 7・7・BAR | REG BONUS |

ボーナス種別は、ボーナス図柄が揃った時点で確定表示する。

ボーナス入賞ゲームは演出上の入賞表示として扱い、追加BETは消費しない(6-1参照)。

### 13-3. 中段CHERRY + BIG時の例外

中段CHERRY + BIG(PREMIUM_BIG)成立時は、中段CHERRY停止 → 2枚払い出し → 次ゲームで7・7・7を自動表示、という流れになる。

PREMIUM_BIGは内部的にBIG確定が既知であるため、JACK LAMP点灯時の表現も最強プレミア扱い(LAMPレインボー等)とすることが推奨される。

## 14. NEON COUNTDOWN

BIG後に発生するRUSH突入チャレンジ。

### 14-1. 発生契機

- 通常時のNORMAL_BIG消化後に必ず発生

## 15. MIDNIGHT RUSH

MIDNIGHT RUSHは、ボーナスと継続判定を繰り返す連チャンモードである。

### 15-1. 基本構造

```text
RUSH突入
 ↓
1セット目 ボーナス確定 ← 初回保証セット
 ↓
BIG or REGを消化
 ↓
ボーナス中に1G連発生?
 ├─ YES → 継続抽選フェーズをスキップして次セットへ
 └─ NO  → 継続抽選フェーズへ
 ↓
5G間の演出
 ↓
6G目レバー演出で結果告知
 ├─ JACK LAMP点灯 → 次セットへ
 └─ 点灯なし → RUSH終了
```

### 15-2. 初回セット保証

**RUSH突入時は、継続抽選を挟まず必ず1セット目のボーナスから開始する。**

これにより、RUSH突入時のプレイヤー体験を保証する。実装時、「RUSH突入 → 継続抽選 → 成功したらボーナス」の順で作ると初回保証が機能しなくなるため、必ず「RUSH突入 → 1セット目ボーナス確定」とする。

---

## 16. RUSH中ボーナス仕様

MIDNIGHT RUSH中は、各セット開始時にボーナスが確定する。

ボーナス種別はBIGとREGの1:1振り分けとする。

| ボーナス | 振り分け | 払い出し | 純増 |
|---|---:|---:|---:|
| BIG | 50% | 168枚 | 132枚 |
| REG | 50% | 56枚 | 44枚 |

### 16-1. 平均値

- 平均ボーナス払い出し:112枚
- 平均ボーナス純増:88枚
- 平均セット数:1 ÷ (1 - 0.70) = 約3.33セット(初回保証セット込み)
- 1RUSH平均純増:88枚 × 3.33セット = 約293枚

実際の純増・機械割はシミュレーションで最終確認する。

### 16-2. RUSH中REG後の処理

RUSH中REG後は通常時へ戻らず、以下のいずれかへ移行する。

- ボーナス中に1G連発生 → 継続抽選フェーズをスキップして次セットへ
- 1G連なし → RUSH継続抽選フェーズへ

通常時REG後との混同を避けるため、bonusContextで管理する(25章参照)。

---

## 17. RUSH継続率の考え方

MIDNIGHT RUSHの継続率は、ボーナス消化中の1G連を含めてトータル70%とする。

### 17-1. 構成内訳

- トータル継続率:70%
- 1G連経由の継続:約7%
- 1G連なし時の内部継続抽選:約67.7%

### 17-2. 計算式

```
トータル継続率 = 1G連発生率 + (1 - 1G連発生率) × 内部継続抽選成功率

0.70 = 0.07 + 0.93 × 内部継続抽選成功率
内部継続抽選成功率 = 約0.677
```

### 17-3. 1G連発生率の数式

1G連発生率は、ボーナス消化中の各ゲームで毎ゲーム1/110を判定するため、ボーナス1回あたりの発生率は以下で算出する。

```
1G連発生率 = 1 - (109/110)^消化G数
```

| ボーナス | 消化G数 | 計算 | 発生率 |
|---|---:|---|---:|
| BIG | 12G | 1 - (109/110)^12 | 約10.4% |
| REG | 4G | 1 - (109/110)^4 | 約3.6% |
| 平均 | 8G | 1 - (109/110)^8 | 約7.0% |

BIGとREGがRUSH中1:1振り分けのため、平均消化Gは8Gとなる。

## 18. 1G連システム

ボーナス消化中に1G連フラグを引いた場合、現在のボーナスを最後まで消化した後、継続抽選フェーズを経由せずに次セットへ移行する。

### 18-1. 1G連抽選確率

全設定共通 1/110、1ゲームあたり。

### 18-2. 1G連の演出パターン

1G連抽選に当選したゲームでは、以下のいずれかの演出を出す。

| 演出パターン | 当選時の振り分け | 発生確率(ボーナス1Gあたり) |
|---|---:|---:|
| 通常1G連演出 | 約96.7% | 約1/114 |
| 中段CHERRY演出 | 約3.3% | 約1/3300 |

中段CHERRY演出は1G連確定の最強パターンとして扱う(11-3参照)。

### 18-3. 連鎖

1G連発生後、次セットのボーナス中でも再び1G連抽選を行う。理論上は連鎖可能だが、Webゲームの安定性を考え、内部的には最大10連鎖までを目安とする。

### 18-4. 実装上の注意

- 1G連抽選はボーナス消化中の各ゲームで毎ゲーム判定
- 1G連発生時は専用エフェクトを出す
- 1G連発生時は継続抽選フェーズをスキップする
- 中段CHERRY演出は1G連当選時の演出パターンの一つであり、1G連発生率自体を変えるものではない

---

## 19. RUSH継続抽選フェーズ

MIDNIGHT RUSH中、ボーナス消化中に1G連が発生しなかった場合に移行する。

### 19-1. 仕様

- BET不要の演出区間
- メダル投入なし
- 払い出しなし
- リプレイなし
- 5G間は液晶演出のみで期待度を示唆
- 6G目のレバー演出で継続結果を告知

### 19-2. 内部抽選

- 1セットごとに約67.7%で継続成功フラグを抽選
- 5G間の演出は、このフラグの結果を見せるためのもの
- フラグの結果は6G目レバー時に告知

### 19-3. 流れ

```text
ボーナス終了
 ↓
1G連なし
 ↓
RUSH継続抽選フェーズ 5G
 ↓
6G目レバー演出
 ├─ JACK LAMP点灯 → RUSH継続、次セットへ
 └─ 点灯なし → RUSH終了、通常時へ
```

### 19-4. 小役風演出について

継続抽選フェーズ中に表示される小役は、実際の払い出し役ではなく、継続期待度を示す演出として扱う。

| 表示役 | 演出上の期待度 |
|---|---|
| 中段CHERRY | 継続確定(内部フラグONのみ出現) |
| BELL | 激熱 |
| JACK | 大チャンス |
| 角CHERRY | チャンス |
| BLUE GEM | 期待度UP |
| REPLAY | 弱チャンス |
| ハズレ | 基本演出 |

### 19-5. 演出と内部フラグの関係

- 内部的に継続フラグONなら、期待度高めの小役表示を優先的に出す
- 内部的に継続フラグOFFなら、期待度低めの小役表示を優先的に出す
- OFFでもまれにレア演出を出し、逆転感を演出する
- 中段CHERRY表示は内部継続フラグON時のみ(11-4参照)

## 20. 天井仕様

### 20-1. 天井条件

通常時500G到達でBIG確定。

### 20-2. 天井恩恵

| 項目 | 内容 |
|---|---|
| 到達条件 | 通常時500G到達 |
| 恩恵 | BIG確定(NORMAL_BIG扱い) |
| RUSH | 確定ではない |
| BIG後 | NEON COUNTDOWNへ移行 |
| COUNTDOWN成功率 | 設定別成功率を使用 |

### 20-3. カウンタリセット条件

以下のいずれかで天井カウンタをリセットする。

- BIG当選時(NORMAL_BIG、PREMIUM_BIG、天井BIGすべて含む)
- REG当選時
- 設定変更時
- RUSH終了時

REGでも天井カウンタはリセットする。これにより、天井狙いが強くなりすぎることを防ぎ、通常時のバランスを保つ。

### 20-4. RUSH中の設定変更時の挙動

RUSH中に設定変更が発生した場合、以下のように処理する。

| 項目 | 挙動 |
|---|---|
| 現在のRUSH | 強制終了 |
| 天井カウンタ | リセット |
| 状態 | 通常時0Gへ |
| 所持コイン | 維持 |

Webゲームでは「設定変更=管理画面操作またはリロード」を想定する。

### 20-5. 天井の調整可能性

500G天井BIG確定は、設定1の機械割を押し上げる要素として強い。シミュレーション結果次第では、以下の案で調整する可能性がある。

| 案 | 内容 | 想定影響 |
|---|---|---|
| 案1(現行) | 500G BIG確定 | 甘め |
| 案2 | 600G BIG確定 | 遊びやすさを残しつつ弱める |
| 案3 | 500G REG以上確定 | BIG確定より弱め |
| 案4 | 500G ボーナス確定、BIG/REG通常振り分け | バランスを取りやすい |

現時点では案1で実装するが、機械割シミュレーション後の調整候補とする。

## 25. 実装時の重要ルール

### 25-1. ボーナス確率の扱い

設定表のBIG確率・REG確率は、単独当選と同時当選を含めた最終実質確率である。

実装では、BIG確率を抽選した後にCHERRY + BIGを別で追加する作り方はしない。

1ゲームごとの抽選で排他的にフラグを決め、最終的なBIG確率・REG確率が設定表に一致するようにする。

### 25-2. BIGフラグの分離管理

BIGは内部的に以下の2フラグに分けて管理する(10-5参照)。

| フラグ | 該当 | 終了後遷移 |
|---|---|---|
| NORMAL_BIG | 単独BIG、CHERRY + BIG、天井BIG | NEON COUNTDOWNへ |
| PREMIUM_BIG | 中段CHERRY + BIG | MIDNIGHT RUSH直行 |

すべてを同じBIGとして扱うと、終了後遷移の判定がブレるため必ず分離する。

### 25-3. ボーナス入賞ゲームのBET処理

通常時ボーナス当選後のボーナス入賞ゲームは、追加BETを消費しない演出区間として扱う(6-1参照)。

実装時、ここで3BETを消費させると以下の問題が起きる。

- プレイヤー視点で「告知後にコインを取られる」不自然さ
- REPLAY後のボーナス入賞ゲームの扱いが複雑化
- 機械割計算とのズレ

### 25-4. ボーナス中のBET消費

BIG中・REG中は各ゲーム開始時に3BETを通常通り消費する(6-2参照)。

実装時、ボーナス中のBET減算を忘れると、BIG純増168枚、REG純増56枚で計算され、機械割が大きく上がるため必ず確認する。

### 25-5. RUSH継続率の扱い

RUSHの継続率70%は、1G連込みのトータル継続率とする。

- 1G連発生時は即継続
- 1G連なし時は内部継続抽選を行う
- 1G連なし時の内部継続抽選成功率は約67.7%

小役風演出は期待度の見せ方であり、実際の払い出しや小役抽選とは分けて管理する。

### 25-6. RUSH初回セット保証

RUSH突入時は、継続抽選を挟まず必ず1セット目のボーナスから開始する(15-2参照)。

「RUSH突入 → 継続抽選 → 成功したらボーナス」の順で実装すると初回保証が機能しなくなるため、必ず「RUSH突入 → 1セット目ボーナス確定」とする。

### 25-7. 通常時REG後とRUSH中REG後の分離

通常時REG後とRUSH中REG後では処理が異なる。

| 発生元 | bonusContext | REG終了後の遷移 |
|---|---|---|
| 通常時REG | NORMAL_REG | 通常時へ |
| RUSH中REG | RUSH_REG | 1G連判定 or RUSH継続抽選フェーズへ |

bonusContextで明確に区別する。

### 25-8. 天井カウンタの扱い

天井カウンタは通常時のみ進む。リセット条件は以下の通り。

- BIG当選(NORMAL_BIG、PREMIUM_BIG、天井BIG)
- REG当選
- 設定変更
- RUSH終了

RUSH中は通常時カウンタを進めない。

### 25-9. Webゲームとしての操作性

プレイヤーが目押しに失敗して損をする仕様にはしない。

図柄を狙う楽しさは演出として残しつつ、内部的には自動制御で成立役を揃える。

### 25-10. 中段CHERRYの実装統一ルール

中段CHERRYは発生する全シーンで「その場面の最強恩恵」に対応する(11章参照)。

| シーン | 処理 |
|---|---|
| 通常時 | PREMIUM_BIGフラグ、BIG + RUSH直行確定 |
| ボーナス中 | 1G連抽選当選時の特別演出パターン、1G連発生率は変えない |
| 継続抽選フェーズ | 内部継続フラグON時の演出表示、払い出しなし |
| NEON COUNTDOWN中 | 出現させない |

### 25-11. プレミア演出の重畳

プレミア演出は通常告知パターンと独立した抽選軸とする。

例として、後ペカ無音 + LAMPレインボーのような組み合わせも発生可能とする。

### 25-12. 入力競合対策

画面クリック、個別ボタン、AUTOの3経路が存在するため、入力競合対策を必須とする。

詳細は28-13を参照。

## 27. 仕様まとめ

NEON JACK v7.2は、通常時の分かりやすさを保ちながら、BIG後のNEON COUNTDOWNとMIDNIGHT RUSHによって、Webスロットらしい出玉感を追加した仕様である。

### 27-1. 設計の3層構造

- 通常時:JACK LAMPが光ればボーナス
- BIG後:NEON COUNTDOWNでRUSH突入抽選
- RUSH中:ボーナスと継続判定をループ

### 27-2. 数値設計の特徴

- 設定1はBIG 1/218、REG 1/272で当たりを軽めにする
- 設定1と設定4は合算が近いが、RUSH突入率で差を出す
- RUSH継続率は全設定共通70%
- 1G連は全設定共通1/110
- RUSH中のBIG/REG比率は1:1
- 目標機械割は101%、107%、110%、115%(シミュレーションで最終調整)

### 27-3. 中段CHERRYの一貫体験

- 中段CHERRYは「見たら最強」が常に成立
- 通常時=BIG+RUSH確定、ボーナス中=1G連確定、継続抽選=継続確定
- プレイヤーの学習コストを最小化

### 27-4. Webゲーム特化

- 目押し不要
- ペカ後は次ゲームのボーナス入賞ゲームで種別判明
- ボーナス入賞ゲームは演出扱いでBET消費なし
- 継続抽選フェーズはBET不要の演出区間
- リール配列は見た目と演出の雰囲気を重視する
- 手動操作と自動押しの両方に対応する

## 28. プレイヤー操作仕様

### 28-1. 基本操作方針

NEON JACKでは、実機スロットの操作感をWeb上で分かりやすく再現するため、通常時・ボーナス中・RUSH中の1ゲームは以下の操作順で進行する。

1. MAXBET
2. レバーON
3. 左リール停止
4. 中リール停止
5. 右リール停止
6. 払い出し・告知・次ゲーム準備(自動)

通常プレイでは、画面内の操作エリアを1回クリックするごとに、現在のゲーム状態に応じた次の操作を実行する。

ただし、6番目の「払い出し・告知・次ゲーム準備」は、右リール停止後に自動で行われる処理とする。

そのため、プレイヤーが1ゲームで直接行うクリック操作は基本的に5回である。

本仕様ではWebゲームとしての遊びやすさを優先し、実機のような長いゲーム間ウェイトは採用しない。

ただし、リール回転・停止・告知・払い出しなどの演出テンポは最低限残す。

### 28-2. 1クリック進行方式

プレイヤーが画面内の操作エリアをクリックするたびに、現在のゲーム状態に応じて次の操作が実行される。

| クリック回数 | 実行される操作 | 内容 |
|---:|---|---|
| 1回目 | MAXBET | 3BETを投入(所持コインから3枚減算) |
| 2回目 | レバーON | 内部抽選・リール回転開始 |
| 3回目 | 左リール停止 | 左リールを停止 |
| 4回目 | 中リール停止 | 中リールを停止 |
| 5回目 | 右リール停止 | 右リールを停止し、役判定・払い出し・告知を行う |

※払い出し処理・ボーナス告知・状態遷移は5回目クリック後に自動実行され、次ゲームのBET待ち状態へ戻る。

REPLAY成立時は、払い出し後に再遊技状態として扱う。次ゲームのBET操作は省略され、自動的に3BET状態へ戻す(所持コイン減算なし)。

### 28-3. 操作ボタン

画面下部に以下の操作ボタンを配置する。

| ボタン | 役割 |
|---|---|
| MAXBET | 3BETを投入 |
| LEVER | レバーON、内部抽選開始 |
| LEFT | 左リール停止 |
| CENTER | 中リール停止 |
| RIGHT | 右リール停止 |
| AUTO | 自動押しON/OFF切り替え |

通常操作では、各ボタンを個別に押してもよいし、画面クリックによる順送り操作でも進行できる。

現在のゲーム状態に対応しないボタンが押された場合、その入力は無効とする。

例:
- BET待ち中にLEFTを押しても反応しない
- 左リール停止待ち中にCENTERを押しても反応しない
- 中リール停止待ち中にLEFTを押しても反応しない
- 払い出し処理中の入力は無効とする

### 28-4. 操作順の固定

本ゲームでは、初心者でも分かりやすく遊べるように、基本停止順は左→中→右の順押し固定とする。

| 状態 | 停止順 |
|---|---|
| 通常時 | 左→中→右 |
| ボーナス中 | 左→中→右 |
| RUSH中ボーナス消化 | 左→中→右 |
| RUSH継続抽選フェーズ | 専用演出操作に従う |
| NEON COUNTDOWN中 | 専用操作仕様に従う |

現在の停止順に対応していない停止ボタンが押された場合、その入力は無効とする。

本ゲームでは順押し固定のため、変則押しによるペナルティや損失は発生しない。

### 28-5. 自動押し機能

手動操作が面倒なプレイヤー向けに、自動押し機能を搭載する。

AUTOボタンをONにすると、現在のゲーム状態に応じて以下の操作を自動で進行する。

1. MAXBET
2. レバーON
3. 左リール停止
4. 中リール停止
5. 右リール停止
6. 払い出し確認
7. 次ゲームへ

自動押しは、手動操作と同じゲーム進行ロジックを使用する。

AUTOはゲーム状態ではなく、操作モードとして管理する。

### 28-6. 自動押しの速度

自動押し中は、ゲームが速く進みすぎないように、各操作に短い待機時間を入れる。

| 操作 | 待機時間の目安 |
|---|---|
| MAXBET → レバーON | 0.2秒 |
| レバーON → 左停止 | 0.4秒 |
| 左停止 → 中停止 | 0.25秒 |
| 中停止 → 右停止 | 0.25秒 |
| 右停止 → 次ゲーム | 0.6秒 |

演出発生時、ボーナス告知時、RUSH突入時、COUNTDOWN発生時は、自動押しを一時停止または遅延させる。

### 28-7. 自動押し停止条件と再開タイミング

以下の場合、自動押しは一時停止する。

| 条件 | 挙動 | 再開条件 |
|---|---|---|
| JACK LAMP点灯 | ボーナス告知を見せるため一時停止 | 最低3秒経過後、自動で次の処理へ |
| BIG/REG開始時 | ボーナス開始演出を見せるため一時停止 | 最低3秒経過後、自動で次の処理へ |
| RUSH突入時 | 突入演出を見せるため一時停止 | 最低5秒経過後、自動で次の処理へ |
| RUSH終了時 | END表示を見せるため一時停止 | 最低3秒経過後、自動で通常時へ |
| 所持コイン不足 | AUTO完全停止 | プレイヤーがコイン補充後、手動で再開 |
| AUTOボタンOFF | 現在の処理完了後に停止 | プレイヤーがAUTOボタン再ON |

連続でボーナスを引いた場合でも、AUTOがONの間は告知最低時間(3秒)経過後に自動的に次ゲームへ進む。プレイヤーが手動でAUTOをOFFにしない限り、毎回クリックを要求することはない。

NEON COUNTDOWNはAUTO中の挙動を別途定義する(28-9参照)。

### 28-8. 自動押し中のボーナス処理

自動押し中にボーナスへ当選した場合、告知タイミングに応じてJACK LAMP点灯演出を表示する。

後ペカの場合は当該ゲームの第3停止後に告知する。

先ペカの場合はレバーON時またはリール始動時に告知する。

ボーナス当選ゲームでは、JACK LAMP点灯までを行い、液晶には「BONUS確定」を表示する(種別は伏せる)。

その後、一定時間ボーナス告知を見せたあと、次ゲーム = ボーナス入賞ゲーム(BET消費なし)へ移行し、ボーナス図柄を自動で揃える。

| ボーナス種別 | 自動停止出目 |
|---|---|
| BIG | 7・7・7 |
| REG | 7・7・BAR |

本仕様では目押し不要のため、手動操作・自動押しを問わず取りこぼしは発生しない。

ボーナス図柄が揃った時点でボーナス種別を確定表示し、BIGまたはREGのボーナス状態へ移行する。

### 28-9. NEON COUNTDOWN中の操作(AUTO一本化版)

NEON COUNTDOWN中は、演出の緊張感を出すため、基本は手動操作を推奨する。

COUNTDOWN中は、通常時とは異なり、BET不要の演出操作として進行する。実際のメダル投入・払い出し・小役抽選は行わない。

NEON COUNTDOWNの操作は以下の9段階で進行する。

| 操作段階 | 操作内容 | 表示 |
|---:|---|---|
| 1回目 | MAXBET + レバーON相当 | カウント8 |
| 2回目 | 左リール停止相当 | カウント7 |
| 3回目 | 中リール停止相当 | カウント6 |
| 4回目 | 右リール停止相当 | カウント5 |
| 5回目 | MAXBET + レバーON相当 | カウント4 |
| 6回目 | 左リール停止相当 | カウント3 |
| 7回目 | 中リール停止相当 | カウント2 |
| 8回目 | 右リール停止相当 | カウント1 |
| 9回目 | ラストMAXBET + レバーON相当 | 最終判定 |

途中でJACK LAMPが点灯した場合は、RUSH突入成功となる。最後まで点灯しなかった場合でも、まれに復活ペカ演出を発生させる。

#### 28-9-1. AUTO中のNEON COUNTDOWN動作(一本化)

AUTO ON中にNEON COUNTDOWNへ移行した場合、以下のフローで進行する。

```text
NEON COUNTDOWN開始
 ↓
5秒間の演出待機(プレイヤーに手動切替の猶予を与える)
 ├─ 5秒以内に手動操作あり → 手動進行へ切替 (AUTOは一時停止)
 └─ 5秒以内に操作なし    → AUTO状態を維持して自動進行
 ↓
COUNTDOWN進行
 ├─ 成功告知・復活演出時 → AUTO一時停止、演出を見せる
 └─ 通常進行 → 各段階0.5秒ごとに自動進行
 ↓
COUNTDOWN終了
 ├─ RUSH突入       → RUSH中AUTOへ移行
 └─ RUSH非突入     → 通常時AUTOへ戻る
```

この仕様により、AUTO中のCOUNTDOWN挙動が一本化され、「途中で止まる」「常に自動進行する」のどちらかで迷う実装にならない。

### 28-10. RUSH継続抽選フェーズ中の操作

RUSH継続抽選フェーズはBET不要の演出区間である。

5G間は液晶演出を進行させ、6G目のレバー演出で継続結果を告知する。

| 操作段階 | 内容 |
|---|---|
| 1〜5G目 | クリックごとに期待度演出を進行 |
| 6G目 | レバー演出でJACK LAMP点灯判定 |
| 継続成功 | 次セットのボーナスへ移行 |
| 継続失敗 | RUSH終了演出後、通常時へ戻る |

この区間では実際のBET、払い出し、小役抽選は行わない。

表示される小役は演出上の期待度表示として扱う。

### 28-11. ウェイト仕様

本ゲームはWebゲームとしての遊びやすさを優先し、実機スロットに存在するような長い強制ウェイトは採用しない。

レバーON後、リールは即座に回転を開始するものとし、プレイヤーは必要以上に待たされない。

ただし、ゲームとしての演出感を保つため、以下の最低演出時間を設ける。

| 処理 | 内容 |
|---|---|
| レバーON後 | リールは即時回転開始 |
| リール停止 | 停止操作後、対象リールを即時停止 |
| 払い出し | 払い出し表示を短時間表示 |
| 告知演出 | JACK LAMP点灯や効果音を一定時間表示 |
| RUSH・COUNTDOWN演出 | 演出を見せるため一時停止または遅延 |

本仕様では、実機のようなゲーム間ウェイトは再現しない。

ただし、自動押し中はゲームが速く進みすぎないよう、操作ごとに短い待機時間を設ける。

### 28-12. 実装上の状態管理

プレイヤー操作は、以下のゲーム状態で管理する。

| 状態名 | 内容 |
|---|---|
| WAIT_BET | BET待ち |
| WAIT_LEVER | レバーON待ち |
| WAIT_STOP_LEFT | 左リール停止待ち |
| WAIT_STOP_CENTER | 中リール停止待ち |
| WAIT_STOP_RIGHT | 右リール停止待ち |
| PAYOUT | 払い出し処理中 |
| BONUS_NOTICE | ボーナス告知中(BONUS確定表示) |
| BONUS_ENTRY | ボーナス入賞ゲーム(BET消費なし、図柄自動揃え) |
| BONUS_GAME | ボーナス消化中 |
| COUNTDOWN | NEON COUNTDOWN中 |
| RUSH_JUDGE | RUSH継続抽選フェーズ |
| RUSH_RESULT | RUSH結果告知 |

AUTOは状態名ではなく、以下のような操作モードとして別管理する。

| 管理項目 | 内容 |
|---|---|
| autoMode | 自動押しON/OFF |
| isProcessing | 処理中フラグ(入力競合対策) |
| currentState | 現在のゲーム状態 |
| currentMode | 通常時、ボーナス中、RUSH中、COUNTDOWN中などの遊技モード |
| bonusContext | 通常時ボーナス、RUSH中ボーナスなどの発生元 |

ボーナス消化は通常時のボーナスでもRUSH中のボーナスでも、基本的には同じBONUS_GAME状態を使用する。

ただし、ボーナス終了後の移行先が異なるため、bonusContextで発生元を管理する。

| bonusContext | ボーナス終了後の移行先 |
|---|---|
| NORMAL_BIG | NEON COUNTDOWNへ |
| NORMAL_REG | 通常時へ |
| CEILING_BIG | NEON COUNTDOWNへ |
| PREMIUM_BIG | MIDNIGHT RUSHへ |
| RUSH_BIG | 1G連判定 or RUSH継続抽選フェーズへ |
| RUSH_REG | 1G連判定 or RUSH継続抽選フェーズへ |

これにより、手動操作と自動押しを同じゲーム進行ロジックで扱えるようにする。

### 28-13. 入力競合対策

画面クリック、個別ボタン、AUTOの3経路が同時に存在するため、入力競合対策を実装する。

#### 28-13-1. 基本ルール

- 入力受付は、現在のゲーム状態 `currentState` ごとに1回のみ有効とする
- 処理中フラグ `isProcessing` が `true` の間は、画面クリック・個別ボタン入力・AUTO入力をすべて無効化する
- AUTO中に手動入力があった場合は、手動入力を優先する
- 手動入力後もAUTOがONの場合は、現在の処理完了後にAUTO進行を再開する
- 払い出し中、告知演出中、リール停止処理中は追加入力を受け付けない

#### 28-13-2. 状態別の入力ロック

| 状態 | 入力受付 |
|---|---|
| WAIT_BET | MAXBETまたは画面クリック |
| WAIT_LEVER | LEVERまたは画面クリック |
| WAIT_STOP_LEFT | LEFTまたは画面クリック |
| WAIT_STOP_CENTER | CENTERまたは画面クリック |
| WAIT_STOP_RIGHT | RIGHTまたは画面クリック |
| PAYOUT | すべて無効 |
| BONUS_NOTICE | すべて無効(最低告知時間後に自動遷移) |
| BONUS_ENTRY | 画面クリックまたはAUTO(リール停止演出のみ) |
| BONUS_GAME | 通常の停止順入力 |
| COUNTDOWN | COUNTDOWN専用操作 |
| RUSH_JUDGE | 画面クリックまたはAUTO |

#### 28-13-3. 連打対策

リール停止処理中など、内部処理に時間がかかる場合は `isProcessing = true` を立てて、すべての入力を無効化する。処理完了後に `isProcessing = false` に戻す。

これにより、連打によるダブル発火や状態の不整合を防ぐ。

## 29. 所持コイン管理

### 29-1. 所持コイン表示

画面に常に所持コイン枚数を表示する。

### 29-2. BET時の減算

| 状態 | 処理 |
|---|---|
| 通常時BET | 所持コインから3枚減算 |
| BIG中BET | 所持コインから3枚減算 |
| REG中BET | 所持コインから3枚減算 |
| REPLAY成立後の次ゲーム | 自動BETとし、所持コインを減算しない |
| ボーナス入賞ゲーム | BET消費なし(6-1参照) |
| NEON COUNTDOWN中 | BET不要のため減算なし |

### 29-3. 払い出し時の加算

各役の払い出し枚数を所持コインに加算する。

| 役 | 加算枚数 |
|---|---:|
| BELL | 14枚 |
| JACK | 10枚 |
| BLUE GEM | 8枚 |
| 角CHERRY | 2枚 |
| 中段CHERRY | 2枚 |
| ボーナス図柄揃い(ボーナス中) | 14枚/回 |

REPLAYは払い出しではなく再遊技として扱うため、所持コインの増減はない。

### 29-4. 所持コイン不足時の処理

所持コインが3枚未満の場合、通常時・ボーナス中の次ゲームは開始できない。

| 状況 | 挙動 |
|---|---|
| 手動操作中にコイン不足 | BET操作を無効化、コイン不足表示 |
| AUTO中にコイン不足 | AUTOを停止、コイン不足表示 |
| ボーナス消化中にコイン不足 | 通常はありえないが、保険として処理停止 |

### 29-5. コイン補充

Webゲームとして、デバッグ用または初期所持コイン設定として、コイン補充機能を持つ。

| 補充方法 | 用途 |
|---|---|
| 初期コイン | ゲーム開始時の初期値(例:1000枚) |
| デバッグ補充 | 開発時のテスト用 |
| 設定変更時 | 所持コインは維持(20-4参照) |

実運用での課金やアイテム購入は本仕様の範囲外とする。
